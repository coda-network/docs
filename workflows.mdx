# Supported workflows

Coda is designed to add an authorization decision before four common digital
asset interactions. Each workflow uses the same decision model while preserving
the responsibilities of your existing systems.

## Wallet creation

Evaluate whether a wallet may be created for an institutional customer or use
case before provisioning begins.

## Signing authorization

Evaluate a proposed signing action before the signer or signing workflow acts
on it.

## Deposit crediting

Evaluate an observed deposit before the institution credits the corresponding
customer or account.

## Withdrawal processing

Evaluate a requested withdrawal before funds are released or a signing action
is initiated.
